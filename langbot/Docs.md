# LangBot

 ## 功能特性

  - 🤖 **多平台支持**：微信、QQ、飞书、钉钉、Discord、Telegram、Slack 等
  - 🧠 **多模型支持**：OpenAI、DeepSeek、Moonshot、Anthropic、xAI、智谱AI、文心一言、讯飞星火、通义千问等
  - 🔧 **LLMOps 支持**：Dify、阿里云百炼、n8n 工作流
  - 🧩 **插件系统**：丰富的插件扩展机制
  - 🌐 **WebUI 管理**：可视化配置界面，无需手动编辑配置文件
  - 🛡️ **稳定性功能**：访问控制、限速、敏感词过滤、多流水线配置

## 安装

  1. 添加此仓库到 Home Assistant 的 Add-on Store
  2. 安装 "LangBot"
  3. 启动插件
  4. 访问 `http://你的HA地址:5300` 进行初始化配置

  ## 配置

  ### 首次启动

  首次启动后，访问 WebUI（端口 5300）创建管理员账号并进行配置：
  
  1. **配置对话模型**：在 WebUI 中添加 LLM API 密钥
  2. **配置消息平台**：设置微信、QQ、飞书等平台接入
  3. **配置流水线**：设置不同场景的处理流程

  ### 数据持久化

  - **数据目录**：`/config/langbot/data`（包含数据库、配置文件）
  - **插件目录**：`/config/langbot/plugins`（安装的插件）

  ### 端口说明

  - **5300**：WebUI 管理界面（必须）
  - **2280-2290**：OneBot 协议消息平台适配器反向连接端口（用于 QQ 等）

  ## 使用示例

  ### 接入 QQ（使用 NapCat）

  1. 在 LangBot WebUI 中启用 `aiocqhttp` 适配器
  2. 配置 Token 和端口（默认 2280）
  3. 在 NapCat 中配置 WebSocket 客户端连接到 `ws://你的HAIP:2280/ws`
  4. 设置相同的 Token

  ### 接入微信（使用 GeweChat）

  1. 在 LangBot 配置中启用 `gewechat` 适配器
  2. 配置 GeweChat 服务地址和回调地址
  3. 扫码登录微信

  ## 注意事项

  1. **首次启动**：需要访问 WebUI 完成初始化，创建管理员账号
  2. **网络要求**：部分平台（如微信、QQ）需要外部网络访问能力
  3. **端口冲突**：确保 5300 和 2280-2290 端口未被占用
  4. **数据备份**：定期备份 `/config/langbot/data` 目录
  5. **插件依赖**：重新创建容器后，可能需要重新安装插件依赖
